# BLM用户手册

## 1. 适用范围

本文档面向当前浏览器版 BLM 用户，说明如何在同一个工作区中完成新建文档、打开文档、复制文档、合并文档、查看回收站，以及使用内置手册。

## 2. 启动方式

在项目根目录执行：

```bash
python blm.py
```

启动后在浏览器中打开本地地址，例如：

```text
http://127.0.0.1:8888
```

## 3. 主界面说明

顶部工具栏包含以下按钮：

- 新建
- 打开
- 保存
- 复制
- 合并
- 导出
- 使用手册

左侧为建模导航区，右侧为编辑区与预览区。

![工作区编辑主界面](screenshots/06_workspace_editor.png)

### 3.1 使用手册入口

1. 点击顶部工具栏里“导出”后面的“使用手册”按钮。
2. 左侧先选择要阅读的文档：设计文档、测试用例或用户手册。
3. 在左侧目录中点击章节，可直接跳转到正文对应位置。

补充说明：

- 使用手册是独立阅读态，不显示“业务域 / 流程 / 数据 / 预览”这排建模页签。
- 页面顶部只保留当前文档标题，不再单独展示高占用说明区。
- 左侧用于切换文档和目录，中间为正文阅读区，截图直接在正文中查看。

![使用手册入口与阅读页](screenshots/09_manual_tab.png)

## 4. 新建文档

1. 点击“新建”。
2. 输入文档名称。
3. 点击“创建”。
4. 系统会在工作区创建对应文档，并自动进入编辑状态。

说明：

- 新建文档后会生成一个默认主流程。
- 创建完成后，文档已经进入工作区，不需要再选择保存位置。

## 5. 打开文档

1. 点击“打开”。
2. 在“工作区文档”页签中选择目标文档。
3. 点击文档条目即可打开。

打开弹窗包含两个页签：

- 工作区文档
- 回收站

在工作区列表中，每个文档条目右侧还提供：

- 历史：查看该文档的历史快照
- 删除：将文档放入回收站

![打开文档弹窗](screenshots/05_open_dialog.png)

## 6. 保存与复制

### 6.1 保存

- 点击“保存”会保存当前文档。
- 如果业务域名称被修改，系统会按新的名称保存工作区文档。
- 若目标名称已存在，会提示是否覆盖同名文档。

### 6.2 复制文档

1. 点击“复制”。
2. 输入一个新的文档名称。
3. 点击“确认复制”。

注意：

- 复制不是“另存到其他目录”，而是在当前工作区复制出一个新文档。
- 复制时名称不能冲突。

## 7. 合并文档

### 7.1 适用场景

适合两个人在同一工作区里分别维护两个文档，定期把它们合成一个新文档。

### 7.2 操作步骤

1. 点击“合并”。
2. 系统会打开“合并文档”弹窗。
3. 在左侧文档、右侧文档两个下拉框中选择要合并的文档。
4. 点击“确认合并”。

系统行为如下：

- 如果没有冲突，会直接生成一个新的合并文档。
- 如果存在冲突，会先展示冲突项。
- 处理完所有冲突后，再次点击“确认合并”即可完成。

![合并文档弹窗](screenshots/07_merge_dialog.png)

### 7.3 合并结果

- 合并结果默认以新文档形式保存到工作区。
- 合并后的文档标题与业务域名称会保持一致。

![合并结果示意](screenshots/08_merge_result.png)

## 8. 回收站与历史版本

### 8.1 历史版本

- 每次覆盖保存前，系统会自动保留旧版本快照。
- 点击“打开”后，在工作区文档条目的“历史”按钮中查看。
- 恢复历史版本前，系统会先为当前版本再保留一个快照。

### 8.2 回收站

- 删除文档时不会直接物理删除，而是进入回收站。
- 点击“打开”，切换到“回收站”页签可查看。
- 点击“恢复”后，文档会回到工作区。

## 9. 未保存修改保护

当文档有未保存修改时：

- 切换文档会弹出确认提示。
- 关闭页面会触发浏览器离开提醒。

建议：

- 重要修改完成后及时保存。
- 大改动前可以主动复制出一个新版本。

## 10. 导出

点击“导出”后，系统会导出两类文件：

- 当前文档 JSON
- 当前文档 Markdown

其中 Markdown 内容来自当前文档结构化数据的派生结果。

## 11. 常见问题

### 11.1 点击“合并”没有反应怎么办？

- 先刷新页面，确保浏览器拿到最新前端代码。
- 确认工作区中至少存在两个文档。
- 如果仍有问题，查看浏览器控制台是否有前端报错。

### 11.2 为什么删除后文档还能找回？

因为系统默认启用了回收站，删除操作是“移入回收站”，不是直接彻底删除。

### 11.3 为什么保存前旧内容还能恢复？

因为系统会在覆盖保存前自动生成历史快照，用于误改、误覆盖后的恢复。

## 13. 2026-04 升级补充

### 13.1 流程组怎么用

- 在流程编辑页中，“流程组”与“业务子域”并列填写。
- 业务子域负责第一层归类，流程组负责把同一子域下的多个细粒度流程再做一层分组。
- 如果暂时不需要分组，可以留空；系统会在左侧目录中显示为“未分组流程”。

### 13.2 节点、用户操作步骤、编排任务

- “任务”现在统一称为“节点”。
- “步骤”现在统一称为“用户操作步骤”，用于描述界面上的点击、查看、提交等用户行为。
- 如果需要继续下钻研发实现，可以在节点中点击“进入编排任务”，维护服务编排、查询来源、目标服务和备注。

### 13.3 使用手册打不开时

- 如果进入“使用手册”后看到提示要求重启 BLM 服务，通常表示浏览器页面已经更新，但本地 Python 服务还是旧版本。
- 关闭旧的 `blm.py` 进程后重新启动，再刷新浏览器页面即可。
