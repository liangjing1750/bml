# BLM测试用例

## 1. 目标

本文档记录当前浏览器版 BLM 的核心回归范围，用于验证工作区编辑、确认合并、回收站恢复、未保存修改保护，以及使用手册阅读页的关键行为。

## 2. 测试分层

### 2.1 Python 单元测试

- 覆盖后端文档迁移、工作区存储、回收站、历史快照、合并分析、文档接口。
- 命令：

```bash
python -m unittest discover -s tests -v
```

### 2.2 浏览器端到端测试

- 使用 Playwright 验证真实 UI 交互。
- 命令：

```bash
cd tools/e2e
npm run test:e2e
```

界面截图可通过以下脚本刷新：

```bash
cd tools/e2e
npm run capture:docs
```

## 3. 核心回归用例

### 3.1 打开文档

| 编号 | 场景 | 预期结果 |
|---|---|---|
| OPEN-01 | 点击“打开” | 弹出工作区文档 / 回收站双页签弹窗 |
| OPEN-02 | 在工作区列表点击文档 | 当前文档切换成功，标题栏显示文档名 |
| OPEN-03 | 有未保存修改时切换文档 | 先弹确认框，取消后仍停留在当前文档 |

### 3.2 保存与复制

| 编号 | 场景 | 预期结果 |
|---|---|---|
| SAVE-01 | 新建文档后保存 | 工作区生成 `.json` 和 `.md` |
| SAVE-02 | 保存已存在文档 | 原文档被覆盖，旧版进入历史快照 |
| SAVE-03 | 复制文档为新名称 | 生成新文档，不覆盖原文档 |
| SAVE-04 | 复制文档使用已存在名称 | 被拒绝并提示更换名称 |

### 3.3 回收站恢复

| 编号 | 场景 | 预期结果 |
|---|---|---|
| REC-01 | 删除工作区文档 | 文档进入回收站，不直接物理删除 |
| REC-02 | 打开回收站 | 可看到文档名和删除时间 |
| REC-03 | 回收站恢复 | 文档回到工作区并可重新打开 |
| REC-04 | 查看历史版本 | 可列出保存前快照 |
| REC-05 | 恢复历史版本 | 当前文档回退到目标快照，恢复前先保留当前快照 |

### 3.4 确认合并

| 编号 | 场景 | 预期结果 |
|---|---|---|
| MERGE-01 | 点击“合并” | 合并弹窗打开，左右下拉框可选择两个不同文档 |
| MERGE-02 | 左右选择同一文档 | 阻止继续并提示不能选择同一文档 |
| MERGE-03 | 点击“确认合并”且无冲突 | 自动生成新的合并文档并切换过去 |
| MERGE-04 | 点击“确认合并”且有冲突 | 展示冲突项，不允许直接完成 |
| MERGE-05 | 完成冲突裁决后再次确认 | 合并成功，生成结果文档 |

![确认合并弹窗示意](screenshots/07_merge_dialog.png)

### 3.5 使用手册入口

| 编号 | 场景 | 预期结果 |
|---|---|---|
| MANUAL-01 | 点击工具栏“使用手册” | 页面切换为独立阅读布局 |
| MANUAL-02 | 进入使用手册后查看顶部 | 不再显示“业务域 / 流程 / 数据 / 预览”页签 |
| MANUAL-03 | 选择“用户手册” | 标题与正文切换成功，正文截图可见 |
| MANUAL-04 | 点击左侧目录项 | 页面滚动到正文对应位置 |

补充验证点：

- 阅读页顶部只保留紧凑标题，不再出现高占用说明区。
- 左侧用于切换文档和目录，正文区直接显示截图，不依赖右侧缩略图栏。

![使用手册入口示意](screenshots/09_manual_tab.png)

### 3.6 未保存修改保护

| 编号 | 场景 | 预期结果 |
|---|---|---|
| GUARD-01 | 修改后切换文档 | 弹出确认提示 |
| GUARD-02 | 修改后关闭页面 | 浏览器触发离开提醒 |
| GUARD-03 | 修改后删除当前文档 | 先确认，取消时保持当前文档 |

## 4. 本轮重点回归

### 4.1 合并按钮可用性

- 防止旧的路径模式状态残留导致“点击合并没有反应”。
- 验证左右下拉框和“确认合并”按钮在浏览器版可正常完成整条流程。

### 4.2 使用手册简洁布局

- 验证进入手册后主建模页签被隐藏。
- 验证阅读区没有高占用说明区。
- 验证手册截图以内联方式展示在正文中。

### 4.3 文档同步维护

- 设计文档、测试用例、用户手册三份文档必须存在。
- 文档必须引用关键截图并覆盖当前主功能关键词。

## 5. 维护要求

出现以下变化时，必须同步补测试或更新用例说明：

- 打开 / 保存 / 复制 / 删除流程变化
- 合并规则、冲突裁决方式变化
- 回收站或历史恢复流程变化
- 使用手册入口或阅读布局变化
