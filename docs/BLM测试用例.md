# BLM测试用例

## 1. 目标

本文档记录当前浏览器版 BLM 的核心回归范围，用于验证工作区编辑、确认合并、回收站恢复、未保存修改保护，以及使用手册阅读页的关键行为。

## 2. 测试分层

### 2.1 Python 单元测试

- 覆盖后端文档迁移、工作区存储、回收站、历史快照、合并分析、文档接口。
- 命令：

```bash
python -m unittest discover -s tests -v
```

### 2.2 浏览器端到端测试

- 使用 Playwright 验证真实 UI 交互。
- 命令：

```bash
cd tools/e2e
npm run test:e2e
```

界面截图可通过以下脚本刷新：

```bash
cd tools/e2e
npm run capture:docs
```

## 3. 核心回归用例

### 3.1 打开文档

| 编号 | 场景 | 预期结果 |
|---|---|---|
| OPEN-01 | 点击“打开” | 弹出工作区文档 / 回收站双页签弹窗 |
| OPEN-02 | 在工作区列表点击文档 | 当前文档切换成功，标题栏显示文档名 |
| OPEN-03 | 有未保存修改时切换文档 | 先弹确认框，取消后仍停留在当前文档 |

### 3.2 保存与复制

| 编号 | 场景 | 预期结果 |
|---|---|---|
| SAVE-01 | 新建文档后保存 | 工作区生成 `.json` 和 `.md` |
| SAVE-02 | 保存已存在文档 | 原文档被覆盖，旧版进入历史快照 |
| SAVE-03 | 复制文档为新名称 | 生成新文档，不覆盖原文档 |
| SAVE-04 | 复制文档使用已存在名称 | 被拒绝并提示更换名称 |

### 3.3 回收站恢复

| 编号 | 场景 | 预期结果 |
|---|---|---|
| REC-01 | 删除工作区文档 | 文档进入回收站，不直接物理删除 |
| REC-02 | 打开回收站 | 可看到文档名和删除时间 |
| REC-03 | 回收站恢复 | 文档回到工作区并可重新打开 |
| REC-04 | 查看历史版本 | 可列出保存前快照 |
| REC-05 | 恢复历史版本 | 当前文档回退到目标快照，恢复前先保留当前快照 |

### 3.4 确认合并

| 编号 | 场景 | 预期结果 |
|---|---|---|
| MERGE-01 | 点击“合并” | 合并弹窗打开，左右下拉框可选择两个不同文档 |
| MERGE-02 | 左右选择同一文档 | 阻止继续并提示不能选择同一文档 |
| MERGE-03 | 点击“确认合并”且无冲突 | 自动生成新的合并文档并切换过去 |
| MERGE-04 | 点击“确认合并”且有冲突 | 展示冲突项，不允许直接完成 |
| MERGE-05 | 完成冲突裁决后再次确认 | 合并成功，生成结果文档 |

![确认合并弹窗示意](screenshots/07_merge_dialog.png)

### 3.5 使用手册入口

| 编号 | 场景 | 预期结果 |
|---|---|---|
| MANUAL-01 | 点击工具栏“使用手册” | 页面切换为独立阅读布局 |
| MANUAL-02 | 进入使用手册后查看顶部 | 不再显示“业务域 / 流程 / 数据 / 预览”页签 |
| MANUAL-03 | 进入使用手册后查看左侧 | 只显示目录，不显示多文档切换列表 |
| MANUAL-04 | 点击左侧目录项 | 页面滚动到正文对应位置 |
| MANUAL-05 | 展开一级目录 | 二级目录展开，便于继续跳转正文 |

补充验证点：

- 阅读页顶部只保留紧凑标题，不再出现高占用说明区。
- 左侧只保留目录，正文区直接显示截图，不依赖右侧缩略图栏。

![使用手册入口示意](screenshots/09_manual_tab.png)

### 3.6 未保存修改保护

| 编号 | 场景 | 预期结果 |
|---|---|---|
| GUARD-01 | 修改后切换文档 | 弹出确认提示 |
| GUARD-02 | 修改后关闭页面 | 浏览器触发离开提醒 |
| GUARD-03 | 修改后删除当前文档 | 先确认，取消时保持当前文档 |

## 4. 本轮重点回归

### 4.1 合并按钮可用性

- 防止旧的路径模式状态残留导致“点击合并没有反应”。
- 验证左右下拉框和“确认合并”按钮在浏览器版可正常完成整条流程。

### 4.2 使用手册简洁布局

- 验证进入手册后主建模页签被隐藏。
- 验证阅读区没有高占用说明区。
- 验证手册截图以内联方式展示在正文中。

### 4.3 文档同步维护

- 设计文档、测试用例、用户手册三份文档必须存在。
- 文档必须引用关键截图并覆盖当前主功能关键词。

## 5. 维护要求

出现以下变化时，必须同步补测试或更新用例说明：

- 打开 / 保存 / 复制 / 删除流程变化
- 合并规则、冲突裁决方式变化
- 回收站或历史恢复流程变化
- 使用手册入口或阅读布局变化
## 6. 2026-04 升级补充

### 6.1 流程组与节点编排

- 新增回归点：流程编辑页可以录入“流程组”，并在左侧目录按“业务子域 -> 流程组 -> 流程”显示。
- 新增回归点：流程编辑页术语统一为“节点”“用户操作步骤”“编排任务”。
- 新增回归点：编排任务在当前节点编辑区内直接维护，不再依赖右侧二级抽屉。
- 新增回归点：节点编辑区会显示任务级流程图，用于展示编排任务串行链路。
- 新增回归点：切换到“任务级视图”时，顶部流程图会显示整个流程的全局任务链，并按节点分框呈现。
- 新增回归点：从“任务级视图”切回“用户步骤视图”后，步骤操作按钮不应重复插入，也不应出现行内容被挤压遮盖。
- 新增回归点：用户操作步骤支持在任一步骤后插入新步骤，并可通过上移、下移动作调整顺序。

### 6.2 流程四视图工作台

- 新增回归点：流程 Tab 默认进入“全景视图”，以一屏价值流矩阵展示业务全景，不直接打开编辑抽屉。
- 新增回归点：全景视图横轴价值流、纵轴业务域或产品边界、单元格状态和说明都来自文档数据，不允许在渲染层写死。
- 新增回归点：打开全景编辑后不出现右侧抽屉，业务人员直接在矩阵中维护价值流列、业务域行、单元格说明，并调整阶段归属。
- 新增回归点：默认交割全景应以“交割智慧监管平台2期”为主行，以“电子仓单系统”为职责参照行；默认价值流为“会员客户、品种参数、业务办理、风险监管”，入库、在库、出库不再作为一级价值流。
- 新增回归点：价值流矩阵不再截断低层阶段清单，阶段卡片全部显示，并支持在单元格内按行列槽位拖拽摆放。
- 新增回归点：打开全景编辑后，各输入框应显示字段提示；表格较大时可通过滚动和缩放继续编辑。
- 新增回归点：流程四视图顶部不应出现高占用说明区，视图说明应收纳为问号提示；全景编辑开关应固定在顶部工具栏，矩阵滚动后仍可见。
- 新增回归点：问号提示应完整显示在视口内，不应被顶部工具栏、卡片或滚动容器裁切。
- 新增回归点：全景编辑时输入框内容变更不应触发页面跳动；新增行列正文为空，清空正文后不应自动回填默认价值流或业务域名称。
- 新增回归点：交割智慧监管平台阶段应优先落到平台主行，并按“会员客户、品种参数、业务办理、风险监管”归类；阶段卡片保持粗粒度，例如账号管理、仓库管理、质检机构管理、品种参数管理、仓单注册、仓单流转、风险监管，交割预报、仓库/厂库仓单注册等细节进入阶段内流程；同一单元格阅读态阶段卡片不应超过 5 个，仓库主体维护、仓库资质维护、仓房维护、垛位维护、提货地点与点位应聚合为“仓库管理”；电子仓单系统只在品种合约维护、仓单注册注销流转等关键环节显示职责差异。
- 新增回归点：从全景阶段节点下钻到阶段视图后，应显示阶段涉及的流程；涉及多个阶段的流程需要显示涉及阶段数量。
- 新增回归点：流程视图默认以业务指引图展示流程，节点采用竖排文字并按办理顺序向右展开。
- 新增回归点：点击阶段详情图中的流程引用节点，应进入同一份流程实体的流程视图和编辑抽屉。
- 新增回归点：从流程视图切换到角色视图时，应按当前角色展示其参与的流程和任务，并可从角色视图返回流程编辑。

### 6.3 文档迁移

- 旧文档加载后需要自动迁移为 `nodes / userSteps / orchestrationTasks / flowGroup` 新结构。
- 保存后重新加载，`schema_version` 需要升级到当前版本，且业务内容不能丢失。

### 6.4 使用手册兼容提示

- 新版前端连接旧版服务时，不应显示为空文档列表，而应提示“当前服务版本过旧，请重启 BLM 服务”。
- 新版前后端匹配时，`/api/docs` 需要正常返回三份内置文档。
