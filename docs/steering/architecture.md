# Architecture Steering

## 长期约束

- 保持本地优先、文件驱动，不引入必须依赖中心服务才能工作的建模链路。
- 文档格式必须可读、可导出、可迁移，重大模型升级要有兼容路径，不能粗暴打断老文档。
- 前端继续以静态页面加轻量状态管理为主，不为了单个功能引入重型框架。
- 数据结构优先清楚表达业务概念边界，再考虑界面组织，避免为了当前 UI 便利把概念强行耦合。

## 关键原则

- 子域拥有流程，阶段引用流程，不再把流程简单视作阶段私有节点。
- 流程是稳定业务资产，阶段是时间线组织视图，流程组只是流程库中的管理分组。
- 先做兼容层，再做默认切换，最后清理旧模型，不在第一步就移除旧字段。
- 所有大改动优先经过 spec，再进入代码与测试，避免“边做边改口径”。

## 不做什么

- 这一轮不引入 BPMN 网关、泳道、条件表达式等复杂流程编排语义。
- 这一轮不把业务子域升级成完整的独立建模工作台，先保留现有 `subDomain` 语义和视图组织。
- 这一轮不做多人在线协同、实时冲突解决和云端中心化存储。
