# J02 同一流程可被多个阶段引用

## 用户旅程

- `Journey ID`：J02
- `用户角色`：业务架构师
- `入口`：流程-业务阶段视图
- `目标`：让支撑性流程跨阶段复用
- `优先级`：P0
- `是否已自动化`：No

## 关键步骤

1. 用户进入 `流程-业务阶段视图`
2. 用户执行 `把同一已有流程加入两个不同业务阶段`
3. 系统展示 `两个阶段都显示该流程且流程详情只有一份`

## BDD 场景

```gherkin
Feature: 同一流程可被多个阶段引用

Scenario: 同一流程可被多个阶段引用
  Given 业务架构师 已进入 流程-业务阶段视图
  When 业务架构师 执行 把同一已有流程加入两个不同业务阶段
  Then 系统应展示 两个阶段都显示该流程且流程详情只有一份
```

## TDD 落点

- 先写失败测试，优先覆盖这条旅程的主断点
- 如果有界面，优先落浏览器/E2E 测试
- 如果涉及状态或产物回写，补接口或路由回归
- 修复后至少回跑当前场景和强相关回归

## 当前缺口

- 当前模型中一个流程只能有一个 `stageId`，因此从数据结构上就不支持“同一流程被多个阶段引用”。
- 如果强行通过复制流程规避，会导致流程资产分裂，后续修改无法保持一致。

## 验收备注

- 同一流程加入两个阶段后，流程详情只有一份，流程 ID 不变。
- 两个阶段都能看到该流程，但各自可以保留独立的阶段内布局位置。
- 从其中一个阶段移除引用，不会删除流程实体，也不会影响另一个阶段的引用。
