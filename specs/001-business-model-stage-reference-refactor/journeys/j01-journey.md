# J01 阶段视图优先引用已有流程

## 用户旅程

- `Journey ID`：J01
- `用户角色`：业务架构师
- `入口`：流程-业务阶段视图
- `目标`：在阶段中复用已有流程而不是重复创建
- `优先级`：P0
- `是否已自动化`：No

## 关键步骤

1. 用户进入 `流程-业务阶段视图`
2. 用户执行 `在当前阶段点击加入已有流程并选择目标流程`
3. 系统展示 `阶段详情图出现该流程且流程本体不被复制`

## BDD 场景

```gherkin
Feature: 阶段视图优先引用已有流程

Scenario: 阶段视图优先引用已有流程
  Given 业务架构师 已进入 流程-业务阶段视图
  When 业务架构师 执行 在当前阶段点击加入已有流程并选择目标流程
  Then 系统应展示 阶段详情图出现该流程且流程本体不被复制
```

## TDD 落点

- 先写失败测试，优先覆盖这条旅程的主断点
- 如果有界面，优先落浏览器/E2E 测试
- 如果涉及状态或产物回写，补接口或路由回归
- 修复后至少回跑当前场景和强相关回归

## 当前缺口

- 当前“加入阶段”本质上会修改 `process.stageId`，导致流程从原阶段移出，而不是在新阶段中新增一条引用。
- 阶段详情图节点直接基于流程实体生成，缺少“阶段中的流程引用”这一层对象。

## 验收备注

- 加入已有流程后，流程总数不增加。
- 新阶段中出现该流程节点，但流程详情仍指向同一个 `process` 实体。
- 原阶段中该流程仍然存在。
- 如果流程名称后续被修改，两个阶段中的展示都同步更新。
