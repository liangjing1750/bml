# J03 按阶段和按子域双视角浏览同一流程资产

## 用户旅程

- `Journey ID`：J03
- `用户角色`：业务分析师
- `入口`：流程页
- `目标`：在不同视角下查看同一流程而不丢失上下文
- `优先级`：P0
- `是否已自动化`：No

## 关键步骤

1. 用户进入 `流程页`
2. 用户执行 `先按阶段打开流程再切换到按子域视图定位同一流程`
3. 系统展示 `系统保持流程上下文并展示同一流程的唯一详情`

## BDD 场景

```gherkin
Feature: 按阶段和按子域双视角浏览同一流程资产

Scenario: 按阶段和按子域双视角浏览同一流程资产
  Given 业务分析师 已进入 流程页
  When 业务分析师 执行 先按阶段打开流程再切换到按子域视图定位同一流程
  Then 系统应展示 系统保持流程上下文并展示同一流程的唯一详情
```

## TDD 落点

- 先写失败测试，优先覆盖这条旅程的主断点
- 如果有界面，优先落浏览器/E2E 测试
- 如果涉及状态或产物回写，补接口或路由回归
- 修复后至少回跑当前场景和强相关回归

## 当前缺口

- 当前左侧目录主要围绕单一树结构组织，阶段视角和子域视角尚未真正解耦。
- 用户从阶段打开流程后，如果想从子域角度回看同一流程，需要重新回忆路径，缺少稳定的“双视角查看同一流程资产”体验。

## 验收备注

- 从阶段视图打开流程后，切到按子域视图时应仍能定位到同一流程。
- 返回导航、抽屉状态和流程详情应尽量保持连续，不应退回首页。
- 阶段视图和子域视图展示的是同一份流程详情，而不是两份独立数据。
